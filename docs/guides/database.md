# Database Guide

This guide documents the database as implemented in **v0**.

## Role of PostgreSQL

PostgreSQL stores application data for the local-first MVP.

The backend is the only layer that should connect to PostgreSQL. The frontend must use FastAPI routes instead of direct database access.

## SQLAlchemy Decision

The project uses SQLAlchemy 2.x sync ORM style for v0.

Reasons:

- simpler for a first full-stack project
- enough for the local-first MVP
- works cleanly with FastAPI service/repository structure
- can be changed later if async database access becomes necessary

## psycopg 3 Decision

The backend uses `psycopg` 3 as the PostgreSQL driver.

The expected SQLAlchemy URL format is:

```txt
postgresql+psycopg://USER:PASSWORD@HOST:PORT/DB_NAME
```

## Alembic Migration Workflow

Alembic manages database schema changes.

A migration is a versioned change to the database. Migrations let the database schema evolve predictably as models change.

Common commands:

```bash
cd backend
uv run alembic revision --autogenerate -m "describe change"
uv run alembic upgrade head
uv run alembic downgrade -1
```

## One Migration in v0

All of v0 runs on the **single initial migration** (`faff5d463169_create_initial_tables.py` — never hand-edit it). Every later phase built API layers on top of the existing models without altering columns, so no further migration was needed. A new autogenerated Alembic migration is only required when a model's columns actually change.

## Index Guidance

Add indexes where they support obvious future access patterns:

- ticker lookups
- unique settings keys
- unique cache keys
- chat message session lookups

Avoid over-indexing before real query patterns exist.

## Usage Notes (v0)

- `market_data_cache` backs the shared cache-aside layer (`core/cache.py`) used by market data (~2 min quote TTL), news (~15 min), and financials (~1 day).
- `reports`, `chat_sessions`, `chat_messages` are written by the `ai/` and `chat/` modules.
- `stock_notes` and `settings` exist in the schema but have no API layer in v0 (reserved for later).
- In Docker Compose, `alembic upgrade head` runs automatically before the API starts.
