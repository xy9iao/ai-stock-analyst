## Phase 12 — Deployment (active)

**Goal:** a live public demo URL + 2–3 defensible production decisions, cheap and abuse-proof. Stop there — no K8s/Terraform/staging, no auth system, no monitoring stack, no load testing (all deliberately out of scope).

**Approved scope, in build order:**

1. **Anonymous session isolation** (not auth): a `DEMO_MODE` env flag (default off — local behavior unchanged, existing data backfilled to a permanent `local` bucket). When on: FastAPI middleware issues a random `session_id` cookie; holdings/watchlist/reports/chat are bucketed per session (indexed column, repositories filter); idle sessions cleaned up after a 7-day TTL; a "New demo session" reset button in the UI.
2. **Cost defenses — three independent layers:** ① budget hard cap: DeepSeek prepaid credit, no auto-recharge (documented); ② **LLM master switch**: `llm_enabled_until` in the existing `settings` table, admin-token endpoint to enable, checked at the single `llm_client` gateway, **default OFF in demo mode**, enable carries a TTL (default 60 min); ③ per-session caps counted in **LLM calls, not requests** (agent-proof): reports ≤3, chat ≤20 — derived from existing rows, `429` beyond.
3. **Observability:** `llm_calls` table written inside `llm_client` (session, kind, model, tokens, latency, + nullable `route`/`steps` pre-seeded for the agent phase) + structured logs + a `GET /api/stats` aggregate endpoint.
4. **One migration for the whole phase** (session columns + `llm_calls`), autogenerated + reviewed.
5. **Deploy:** Vercel (frontend) + Render free (backend) + Neon free (Postgres). Cold start on the free tier is **accepted**; the deploy guide carries an "application-season upgrade" TODO (paid instance when actively interviewing).
6. **CI/CD story:** existing CI gates every PR; Render/Vercel auto-deploy `main` — "tests gate the merge; the merge is the deploy."
7. **Docs:** `docs/guides/deployment.md`, README demo URL + limits note, decisions.md entry, close v0.
