# Changelog

Build history by phase — the **accurate, frozen record** of what was actually built, newest first. The active phase lives in `docs/roadmap.md`. Per the per-phase handoff rules in `CLAUDE.md`, append a new section here when a phase is finished.

**v0 at a glance (June – July 2026):** Phases 0–11 delivered a local-first AI stock research assistant — holdings/watchlist CRUD → market data (provider abstraction + cache) → news/financials → AI reports (single LLM gateway, compact context injection) → multi-turn chat → Markdown export → UI polish/design system → test suites (43 backend + 28 frontend) → the public README. Plus off-roadmap: one-command Docker dev, the frontend MVP UI, ESLint/CI hardening. v0 was declared **feature-frozen on 2026-07-02**; Phase 12 (deploy-prep) remains.

## Phase 12 — Deployment (done 2026-07-03)

- Built: the demo-hardened deployment layer, closing v0. **Anonymous session isolation** behind a `DEMO_MODE` flag (default off — local use unchanged): cookie middleware (`core/demo_session.py`), an indexed `session_id` bucket on holdings/watchlist/reports/chat (`SessionScopedMixin`; existing rows backfilled to `"local"` via `server_default`), repository-level filtering, 7-day TTL lazy cleanup, and a "New demo session" footer button. **Three-layer LLM cost defense:** ① DeepSeek prepaid balance = budget hard cap (documented) ② a **master switch** stored in the `settings` table (`llm_enabled_until`), toggled via the token-gated `POST /api/admin/llm` (fail-closed 403 when `ADMIN_TOKEN` unset), checked inside the single `llm_client` gateway, default OFF with a TTL-bound enable (60 min) ③ **per-session caps counted in LLM calls** (reports ≤3, chat replies ≤20 → 429), derived from existing rows. **Observability:** one `llm_calls` row per LLM call (tokens, latency, + `route`/`steps` pre-seeded for the agent version) written in the gateway, plus structured logs and a public `GET /api/stats` aggregate endpoint; LLM calls also gained an explicit timeout. One reviewed migration (`b3074aa5c807`). A step-by-step **deploy guide** for Vercel + Render (free) + Neon (free), with the cold-start trade-off accepted and an application-season upgrade TODO.
- Files: `backend/app/core/{demo_session,llm_switch}.py` (new), `config.py` (demo settings), `models/{base,holding,watchlist_item,report,chat,llm_call,__init__}.py`, `modules/{holdings,watchlist,ai,chat}/` (session threading + caps), `modules/ai/llm_client.py` (switch check, timeout, call logging), `modules/{admin,session}/` (new), `main.py`, `alembic/versions/b3074aa5c807_*.py`, `tests/test_demo.py` (9 new; 52 total); `frontend/components/layout/Footer.tsx` + `lib/api/session.ts` (reset button); `docs/guides/deployment.md` (new), `docs/guides/{api,backend,database}.md`, `README.md` (Live Demo section), `.env.example`, `docs/planning/decisions.md` (Decision 009).
- Key decisions: **anonymous sessions instead of an auth system** (the demo needs state isolation + cost control, not identity — scope by requirement); caps counted in **LLM calls, not requests** (stays valid when the agent version multiplies calls per request); the master switch **fails closed** and always expires; observability seeded for pipeline-vs-agent experiments; free tier + cold start accepted over paid/keep-alive (upgrade only while interviewing).

## Phase 5 — News and Financial Data Integration (done 2026-06-30)

- Key decisions: **yfinance** for both news and financials (no new dependency, no key; swap at deployment); **shared `core/cache.py`** instead of per-module cache repositories; news is a **secondary signal** — unknown/empty ticker returns `200` with empty `items` (yfinance can't distinguish "no news" from "bad ticker"), whereas financials `404` on unknown; backend-only (no frontend this phase); no new migration (reuse `market_data_cache`).
