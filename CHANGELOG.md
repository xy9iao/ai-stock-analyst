## Phase 14 â€” Hybrid RAG + Cited Reports (done 2026-07-21)

Reports grounded in retrievable sources: ingested article bodies â†’ hybrid retrieval â†’ claims carrying clickable citations. Built 2026-07-20 â†’ 2026-07-21 in PR #33. Design fixed at kickoff: **no RAG UI mode** â€” retrieval is a fixed step in pipeline reports and a 6th agent tool (`search_documents`), both through one shared retrieval path; the user-visible change is citations appearing.

- **Ingestion** â€” news URLs â†’ trafilatura body extraction â†’ deterministic chunker (1800 chars, 200 overlap, sentence/whitespace boundary snapping with a provable termination guard; `(source_url, chunk_index)` identity keeps citations stable) â†’ OpenAI `text-embedding-3-small` via the single `embeddings_client` seam (`kind='embed'` in `llm_calls`) â†’ `document_chunks` on pgvector. Idempotent per URL; per-doc failures skipped.
- **Hybrid retrieval** â€” pgvector cosine scan (no ANN index â€” defended by corpus-size math) + Postgres FTS pool (GIN expression index) rescored with real BM25 (`rank_bm25`), fused by RRF (k=60, fuse-then-truncate). **Live-fire fix:** `websearch_to_tsquery` ANDs terms â†’ empty lexical pools on natural-language queries; recall-stage queries now OR terms, BM25's IDF supplies precision. Acceptance example on the live corpus: "data center revenue outlook" â†’ hybrid top-8 rescues 3 chunks vector-only misses + 1 chunk lexical-only misses.
- **Cited generation** â€” prompts inject `[chunk:id]`-tagged Sources blocks (same format both paths); the model cites by copying tags; save-time rewrite to numbered links + a `## Sources` section in the stored Markdown (citations survive export, zero frontend changes); deterministic set-membership validation, one corrective retry (pipeline), strip + visible note as the floor; `[unverified]` badge for unsupported claims.
- **The regression gate's first real catch** â€” post-RAG prompts scored **0.9167 vs the 0.917 floor: merge blocked**. Diagnosis: step-limit hits 2/20 â†’ 10/20 (retrieval inflates history; forced answers dropped disclaimers/safety phrasing â€” one "guaranteed" zeroed a case). Fix: agent-path top-k 8â†’5, budget-awareness in the loop prompt, citation-preserving forced-answer message, runner persists imperfect memos. **Rerun: regression 1.000, citation cases 5/5.**
- **Citation eval** â€” 5 `{query, cited_atom}` cases (presence + the cited chunk contains the atom), with authoring rules learned across three live iterations: atoms must be document-only (tickers get sourced from price tools, correctly untagged), uniquely tied to one source context, and may be named in the query (the gate tests attribution, not surfacing).
- **Files** â€” added `models/document_chunk.py`, migrations `c953934595fd` + `80ad439a5932`, `ai/rag/{chunking,retrieval,citations,ingest,embeddings_client}.py`, `eval/{citation_check,citation_cases.json,corpus}.py`, 6 test files; modified loop/tools prompts, `report_generator`, `prompt_builder`, `llm_client` (`model` param), `eval/run.py`, config/env/docker-compose. Suite: 145 backend + 28 frontend.
- **Key decisions** â€” pgvector over a dedicated vector DB (vectors are a column, not a database: one transaction boundary, one query engine for both retrieval paths, $0 infra); FTS-for-recall/BM25-for-ranking split (claim-precision: BM25 genuinely runs); RRF over score normalization; deterministic set-membership citation validation over LLM judging; inline tags over structured output (per-claim degradation beats per-report).
